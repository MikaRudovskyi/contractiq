# ContractIQ — Frontend (React + TypeScript)

Повноцінний інтерфейс системи управління підрядниками. Підключений до backend на C# (.NET) — усі дані і всі дії проходять через реальний API.

## Перед запуском

1. Backend (`ContractIQ.Api`) має бути запущений і слухати на `http://localhost:5080`.
2. Перевір `http://localhost:5080/swagger` у браузері — якщо відкривається, можна запускати фронтенд.

## Встановлення і запуск

```
npm install
npm run dev
```

Відкрий посилання з консолі (типу `http://localhost:5173`).

## Що тепер працює повністю (всі кнопки реальні)

- **Підрядники**: кнопка "Додати підрядника" відкриває форму, реально створює запис через API. Клік по рядку в таблиці відкриває форму редагування (зміна статусу, приміток).
- **Договори**: "Новий договір" — повна форма з вибором підрядника, типу оплати, сум і дат. Клік по рядку — зміна статусу й прогресу виконання.
- **Акти виконаних робіт**: "Новий акт" — форма подання акту по договору. Кнопки ✓/✗ на картках реально викликають backend (прийняття автоматично створює виплату, відхилення вимагає коментар).
- **Виплати**: "Запланувати виплату" — форма створення. Клік по рядку — зміна статусу (зокрема позначення "Оплачено").
- **Документи**: "Завантажити документ" — форма з вибором файлу, типу, терміну дії. Клік по рядку — затвердити/відхилити документ.
- **Експорт** (сторінка підрядників) — реально генерує і завантажує CSV-файл з поточного списку (з урахуванням фільтрів).

## Важливе обмеження — завантаження файлів документів

Backend поки **не має окремого ендпоінту для зберігання файлів** (наприклад, на диску чи в S3/Blob Storage) — лише таблицю з метаданими документа (назва, тип, термін дії, ім'я файлу, розмір).

Тому форма "Завантажити документ" зараз:
- ✅ Реально створює запис у базі даних зі всіма метаданими
- ✅ Показує файл у реєстрі документів з правильною назвою і розміром
- ❌ **Не зберігає сам бінарний файл на сервері** — тільки інформацію про нього

Це позначено жовтим попередженням прямо у формі завантаження, щоб не створювати ілюзію, що файл фізично десь лежить.

**Якщо потрібне справжнє зберігання файлів** — це наступний крок розробки: додати на backend ендпоінт типу `POST /api/documents/upload` з `IFormFile`, який писатиме файл на диск (або в хмару) і повертатиме `storagePath`. Скажи, якщо хочеш, щоб я це зробив.

## Якщо щось не працює

**Форма відкривається, але при збереженні з'являється червоне повідомлення про помилку:**
Це нормальна обробка помилок — текст помилки прийшов прямо від backend (наприклад: "Договір можна створити лише з активним підрядником"). Прочитай повідомлення — воно описує саме бізнес-правило, яке не виконано.

**Випадаючий список підрядників/договорів у формі порожній:**
Перевір, що у тебе є хоча б один підрядник зі статусом "Активний" (для форми договору) або хоча б один активний договір (для форми акту/виплати).
